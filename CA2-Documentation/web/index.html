Introduktion

Vi fik opgaven at udvikle en simpel chat service med enkelt brugergrænseflade. Systemet skulle kunne håndtere mange brugere samtidig og man skulle logge ind, skrive til alle eller én person.

Chat servicen skulle køre på online tilgængelig server og vi fik udleveret en protokol som fortalte hvilke kommandoer chat servicen skulle reagere på.

Scope var at kunne interagere med andre gruppers chat service både på client og server niveau.

Kort brugermanual

For at kunne benytte GUI klienten og logge på vores server, skal du starte vores ca2.client/Run.java

Du kan også benytte telnet 165.227.151.92 8081.

I vores GUI kan man connecte og disconnecte og man kan skrive til én eller flere brugere i privat eller fællesrum.

Arkitektur

Vi valgte at dele Client og Server op i 2 Java projekter. Dette for at de hver især skal kunne deployes adskilleligt.

Vores kode indeholder både Classes, Threads, Sockets og ArrayBlockingQueue.

Design

Serveren fungerer som følgende: Serveren starter op, åbner socket, launcher postman (der håndterer beskeder imellem klienter) og starter et loop der lytter på nye klient forbindelser. Når en klient logger på, bliver han instantieret som ny ClientPocket (som håndterer klientfunktioner), og lagt til en arraylist af ClientPockets.

ClientPocket funger som følgende: ClientPocket får en clientsocket med i sin constructor, lytter på om klienten er logget ind og lytter herefter konstant på om klienten chatter eller logger ud.

Postman fungerer som følgende: Postman nedarver fra Thread og fungerer som en gammeldags telefondame, der parrer beskeder med klienter. Beskederne bliver tilføjet et ArrayBlockingQueue for herefter at blive fordelt simultant.
